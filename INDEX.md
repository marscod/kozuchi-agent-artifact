# Paper Number Index

This file is the canonical map between **every empirical claim in the
paper** and the script / data file / figure / paper section it comes
from. Reviewers can use it as a checklist: pick a row, run the script,
compare its output to the cited paper section.

* "Section" links go into [`paper/main.tex`](paper/main.tex) by `\label{}`
  anchor.  Open the PDF at the same `\autoref{}` to find the printed
  number.
* "Script" links resolve relative to this folder.
* "Output" cells link to the regenerated artifact under
  [`stats/`](stats/) or [`figures/`](figures/).

Run [`reproduce.sh`](reproduce.sh) to regenerate every output below in
one go.

## Provenance trail

Every script writes a JSON summary alongside the LaTeX fragment so a
reviewer can cross-check the input file paths. For example:

```bash
$ python3 -m json.tool stats/xcheck_summary.json | head -10
{
    "selector_family": "simple_weighted_passrate",
    "f2p_weight": 0.3,
    "p2p_weight": 0.7,
    "tie_break": "shortest_patch_raw",
    "expected_count": 500,
    "selected_count": 495,
    "resolved_count": 376,
    "unresolved_count": 117,
    "error_count": 2,
```

Numbers cited in the manuscript are pulled from these JSON files at
build time (the LaTeX uses `\input{stats/...tex}`), so the paper and
the artifact are mechanically consistent.
